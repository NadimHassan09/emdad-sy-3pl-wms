## Ops Checklist to Complete Full Certification

1. **Google Cloud Console**
   - Enable Google Drive API
   - Create OAuth 2.0 Web client
   - Add redirect URI: `https://staging-admin.emdadsy.com/api/integrations/google-drive/callback`

2. **Server `.env`** (do not commit)
   ```env
   BACKUP_GDRIVE_CLIENT_ID=<client-id>.apps.googleusercontent.com
   BACKUP_GDRIVE_CLIENT_SECRET=<client-secret>
   BACKUP_GDRIVE_STARTUP_STRICT=true
   ```

3. **Restart backend**
   ```bash
   cd backend && npm run build
   pm2 restart emdad-wms-backend-staging --update-env
   ```

4. **Connect Drive** — Admin UI → Settings → Backups → Google Drive → **Connect Drive**

5. **Re-run certification**
   ```bash
   node scripts/backup-gdrive-dr-cert.mjs
   ```
   Expect verdict **PASS** with live sync checks passing.

## Bug Fixed During Certification

**OAuth callback returned 403 Forbidden** for unauthenticated Google redirects because class-level `RolesGuard` on `BackupDriveController` did not honor `@Public()`. Fixed in `roles.guard.ts` by skipping role checks when `IS_PUBLIC_KEY` is set.

`BACKUP_GDRIVE_STARTUP_STRICT=false` is set on the staging server to allow the backend to boot while OAuth credentials are being provisioned. Set to `true` (or remove to default strict in production) once credentials are configured.
